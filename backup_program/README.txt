Напишем свою небольшую систему резервного копирования на bash!


Наш скрипт должен обладать следующими возможностями:

1. Брать по указанному пути все файлы, имеющие заданное расширение (например, .db), помещать их в единый архив (например, .zip) и сохранять этот архив по заданному пути;

2. Должна обеспечиваться возможность проверить целостность готовой резервной копии, например, при помощи контрольных сумм;

3. Должна обеспечиваться возможность задать периодичность резервного копирования;

4. Должна обеспечиваться возможность вытеснения самой старой резервной копии, для исключения заполнения диска (как опция). При включении данной опции задаётся число n, если в месте хранения уже существует n резервных копий, самая старая из них удаляется, после чего производится создание актуальной копии.

3. Принимать параметры работы можно через конфигурационный файл или через аргументы командной строки - как вам будет удобнее;
